# Contributing Guidelines

Thank you for considering contributing to this project! 🎉  
We welcome contributions of all kinds—whether it’s fixing a bug, improving documentation, suggesting features, or contributing code.  

Please take a moment to review these guidelines before making a contribution.

---

## 📌 How to Contribute

1. **Fork the repository**  
   Create your own copy of the repository by clicking the "Fork" button.
2. **Create a branch**  
   Use a descriptive name for your branch:
   ```bash
   git checkout -b feature/add-new-component
   git checkout -b fix/typo-in-readme
   ```
3. **Make your changes**
* Follow the project’s coding standards and style guides.
* Add or update tests as needed.
* Update documentation if your changes affect usage.
4. **Commit your changes**
* Write clear, descriptive commit messages:
* Push your branch
  ```bash
  git push origin feature/add-new-component
  ```
5. **Open a Pull Request (PR)**
* Explain your changes and reference any related issues.
* Make sure all CI checks pass before requesting a review.

## ✅ Code Style
* Follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) for commit messages
* Ensure code is formatted according to project style (e.g., dotnet format, eslint, prettier, etc.).
* Write clear and maintainable code with comments where necessary.

## 🧪 Testing
* Run existing tests to ensure your changes don’t break anything.
* Add new tests for new features or bug fixes.
* Ensure all tests pass before submitting a PR.

## 📖 Documentation
* Update README, inline comments, or other docs when adding new features.
* For major features, consider adding usage examples.

## 🐛 Reporting Issues
* Use the Issues tab to report bugs or request features. Provide as much detail as possible:
  * Steps to reproduce
  * Expected vs. actual behavior
  * Screenshots or logs, if applicable

## 💬 Community
* Be respectful and constructive in all discussions.
* Follow our Code of Conduct
* Collaboration and kindness make the project better for everyone.
